# 路线图

## 阶段一：公开项目骨架

目标：把产品边界和协作方式说清楚。

- [x] 创建 README。
- [x] 创建需求说明。
- [x] 创建内容协议。
- [x] 创建发布工作流。
- [x] 创建 Issue 模板。
- [x] 创建前端项目说明。

## 阶段二：第一篇内容上线

目标：用一篇真实文章验证混合模式。

- [x] 将第一篇文章从私人 `raw/publish/` 转换到 `content/articles/`。
- [x] 转换 Obsidian wikilinks。
- [x] 处理 Mermaid 或公开图片资源。
- [ ] 创建文章讨论 Issue。
- [ ] 回填 `discussion.issue` 和 `discussion.url`。

## 阶段三：最小前端站点

目标：提供比 GitHub Markdown 更好的阅读体验。

- [ ] 文章列表页。
- [ ] 文章详情页。
- [ ] 标签展示。
- [ ] Markdown 渲染。
- [ ] Mermaid 渲染。
- [ ] GitHub Issue 讨论链接。

## 阶段四：发布自动化

目标：减少手动复制和漏填字段。

- [ ] 校验 `content/articles/` frontmatter。
- [x] 从私人 `raw/publish/` 生成公开 Markdown。
- [x] 自动转换 Obsidian 链接。
- [ ] 自动复制公开资源。
- [ ] 自动创建或复用 GitHub Issue。
- [ ] 生成发布报告。

## 阶段五：多形态输出

目标：让一份内容派生为多个媒介。

- [ ] RSS。
- [ ] 幻灯片版本。
- [ ] 视频脚本版本。
- [ ] 图文卡片版本。
- [ ] 系列专题页。
